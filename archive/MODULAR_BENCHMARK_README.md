# RFT Modular Benchmark Suite

This is a restructured, modular benchmark suite for testing Resonance Field Theory (RFT) implementations against standard computational methods. The suite is designed to be **parameter-agnostic**, **scalable**, and **CI/CD friendly**.

## Architecture Overview

### Modular Structure
- **`benchmark_controller.py`** - Master controller for running individual or all benchmarks
- **`benchmark_utils.py`** - Shared utilities and configurations
- **`crypto_benchmark.py`** - Cryptographic robustness tests
- **`quantum_benchmark.py`** - Quantum coherence preservation tests
- **`optimization_benchmark.py`** - Non-linear optimization tests
- **`pattern_benchmark.py`** - High-dimensional pattern recognition tests

### Key Benefits

✅ **Performance** - Run only what you need, avoid loading all tests into memory
✅ **Granular Profiling** - See exactly which sub-test is slowest or least stable
✅ **Easier Debugging** - If one test throws, you don't lose other results ✅ **CI/CD Integration** - Run quick subsets on commits, full suite manually ✅ **No Hardcoding** - All parameters are configurable and environment-adaptive ## Quick Start ### List Available Benchmarks ```bash python benchmark_controller.py --list ``` ### Run Individual Benchmarks ```bash # Crypto benchmark only (fast) python benchmark_controller.py --only crypto --scale small # Quantum benchmark with custom parameters python benchmark_controller.py --only quantum --scale medium --n-qubits 3 # Multiple specific benchmarks python benchmark_controller.py --only crypto quantum --scale large ``` ### Run All Benchmarks ```bash # Quick test (Codespaces-friendly) python benchmark_controller.py --all --scale small # Full benchmark suite python benchmark_controller.py --all --scale large ``` ### Direct Benchmark Execution ```bash # Run crypto benchmark directly with custom parameters python crypto_benchmark.py --num-tests 500 --scale medium --output my_results.json # Run quantum benchmark with specific configuration python quantum_benchmark.py --n-qubits 4 --circuit-depth 30 --num-trials 20 ``` ## Scaling Options The `--scale` parameter automatically adjusts test sizes for different environments: - **`small`** - Quick tests for development/CI (10-100 samples) - **`medium`** - Standard tests for regular validation (100-1000 samples) - **`large`** - Comprehensive tests for final reporting (1000-10000 samples) - **`xlarge`** - Maximum tests for research/publication (10000+ samples) ## Individual Benchmark Details ### 1. Cryptographic Robustness (`crypto_benchmark.py`) Tests RFT-based crypto transformations vs AES-256 for differential attack resistance. **Parameters:** - `--num-tests` - Number of plaintext pairs to test - `--scale` - Environment scaling factor **Example:** ```bash python crypto_benchmark.py --num-tests 2000 --scale large ``` ### 2. Quantum Coherence (`quantum_benchmark.py`) Tests RFT-enhanced quantum simulation vs standard simulation under noise. **Parameters:** - `--n-qubits` - Number of qubits in simulation - `--circuit-depth` - Depth of quantum circuit - `--num-trials` - Trials per noise level **Example:** ```bash python quantum_benchmark.py --n-qubits 5 --circuit-depth 50 --num-trials 15 ``` ### 3. Optimization (`optimization_benchmark.py`) Tests RFT-guided optimization vs scipy on Rosenbrock function with constraints. **Parameters:** - `--dimensions` - Problem dimensions to test (space-separated) - `--max-iterations` - Maximum optimization iterations - `--num-restarts` - Number of restart attempts **Example:** ```bash python optimization_benchmark.py --dimensions 10 25 50 100 --max-iterations 2000 ``` ### 4. Pattern Recognition (`pattern_benchmark.py`) Tests RFT pattern recognition vs PCA on high-dimensional data with embedded patterns. **Parameters:** - `--n-samples` - Number of data samples - `--n-features` - Number of features per sample - `--n-patterns` - Number of patterns to embed **Example:** ```bash python pattern_benchmark.py --n-samples 2000 --n-features 500 --n-patterns 8 ``` ## Configuration and Reproducibility ### Random Seeds All benchmarks use `--random-seed` for reproducible results: ```bash python benchmark_controller.py --all --random-seed 12345 ``` ### Output Files Each benchmark saves results to JSON: - `crypto_benchmark_results.json` - `quantum_benchmark_results.json` - `optimization_benchmark_results.json` - `pattern_benchmark_results.json` - `rft_benchmark_suite_results.json` (combined results) ### Custom Output ```bash python crypto_benchmark.py --output my_crypto_results.json ``` ## CI/CD Integration ### GitHub Actions Example ```yaml name: RFT Benchmark CI on: [push, pull_request] jobs: quick-benchmark: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 - name: Setup Python uses: actions/setup-python@v4 with: python-version: '3.9' - name: Install dependencies run: pip install numpy scipy - name: Run quick benchmarks run: python benchmark_controller.py --all --scale small full-benchmark: runs-on: ubuntu-latest if: github.event_name == 'workflow_dispatch' # Manual trigger only steps: - uses: actions/checkout@v3 - name: Setup Python uses: actions/setup-python@v4 with: python-version: '3.9' - name: Install dependencies run: pip install numpy scipy scikit-learn - name: Run full benchmark suite run: python benchmark_controller.py --all --scale large ``` ### Makefile Integration ```makefile # Quick development tests test-quick: python benchmark_controller.py --all --scale small # Standard validation test-standard: python benchmark_controller.py --all --scale medium # Full benchmark suite test-full: python benchmark_controller.py --all --scale large # Individual benchmarks test-crypto: python benchmark_controller.py --only crypto --scale medium test-quantum: python benchmark_controller.py --only quantum --scale medium ``` ## Performance Tips ### For Codespaces/Limited Resources ```bash # Minimal test python benchmark_controller.py --only crypto --scale small --num-tests 50 # Medium test python benchmark_controller.py --only crypto quantum --scale medium ``` ### For High-Performance Systems ```bash # Maximum testing python benchmark_controller.py --all --scale xlarge # Custom high-performance crypto test python crypto_benchmark.py --num-tests 10000 --scale xlarge ``` ### Memory Management - Pattern recognition benchmark processes data in batches - Quantum benchmark scales qubit count and circuit depth appropriately - Optimization benchmark adjusts problem dimensions based on scale ## Dependencies **Required:** - `numpy` - `scipy` - Custom RFT modules: `minimal_true_rft.py`, `corrected_rft_quantum.py` **Optional (for enhanced features):** - `scikit-learn` - For PCA comparison in pattern recognition - Manual fallbacks provided when sklearn unavailable ## Error Handling Each benchmark includes: - Graceful fallbacks for missing dependencies - Input validation and bounds checking - Comprehensive error reporting - Status tracking in controller ## Results Analysis Results include: - Performance metrics (time, accuracy, robustness) - Improvement ratios (RFT vs standard methods) - Statistical validation data - Configuration metadata for reproducibility ## Extending the Suite To add a new benchmark: 1. Create `new_benchmark.py` following the pattern: ```python from benchmark_utils import BenchmarkUtils, ConfigurableBenchmark class NewBenchmark(ConfigurableBenchmark): def run_benchmark(self) -> Dict: # Your benchmark implementation pass ``` 2. Add to `benchmark_controller.py`: ```python self.available_benchmarks['new'] = { 'script': 'new_benchmark.py', 'description': 'Description of new benchmark'
}
```

3. Update CLI arguments as needed

## Migration from Original

The original `external_wins_benchmark.py` has been split into:
- Individual focused benchmarks for modularity
- Shared utilities for code reuse
- Master controller for orchestration
- Configurable parameters instead of hardcoded values

All original functionality is preserved while gaining the benefits of modularity and scalability.
